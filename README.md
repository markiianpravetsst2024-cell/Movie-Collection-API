# 🎬 Movie Collection REST API

Цей проєкт є виконанням практичного домашнього завдання з курсу **FastAPI: від теорії до практики**. Реалізовано повноцінне REST API для керування колекцією фільмів та базою даних режисерів за допомогою сучасного асинхронного фреймворку FastAPI, ORM SQLAlchemy та валідації Pydantic.

---

## 🛠️ Технологічний стек
* **Python 3.10+**
* **FastAPI** — мікрофреймворк для побудови високопродуктивних API.
* **Uvicorn** — швидкий ASGI-сервер.
* **SQLAlchemy (ORM)** — інструмент для роботи з реляційною базою даних через об'єкти Python.
* **Pydantic v2** — валідація вхідних даних, сувора типізація та автоматична генерація схем.
* **SQLite** — вбудована легковагова база даних.

---

## 📁 Архітектура та структура проєкту
Код проєкту суворо розділений на 5 незалежних шарів (файлів) відповідно до критеріїв оцінювання, що запобігає дублюванню логіки:

1. **`database.py`** — Налаштування підключення до SQLite (`engine`, `SessionLocal`), базовий клас `Base`. Містить функцію-генератор `get_db()` з конструкцією `yield` та обов'язковим закриттям сесії у блоці `finally` (Dependency Injection).
2. **`models.py`** — Опис таблиць бази даних за допомогою SQLAlchemy. Створено дві пов'язані моделі: `Movie` (Фільми) та `Director` (Режисери). Зв'язок реалізовано за принципом один-до-багатьох через `ForeignKey` та `relationship`.
3. **`schemas.py`** — Схеми валідації Pydantic. Розділено вхідні моделі (`MovieCreate`, `DirectorCreate`) та вихідні структури відповідей (`MovieResponse`, `DirectorResponse`, `DirectorMoviesResponse`). Усі схеми відповідей містять параметр `from_attributes = True` для коректної серіалізації ORM-об'єктів. Рейтинг фільму валідується в межах від 0.0 до 10.0 за допомогою `Field(ge=0.0, le=10.0)`.
4. **`crud.py`** — Ізольована логіка прямої взаємодії з базою даних. Жоден SQL-запит або сесія (`db.add`, `db.commit`) не викликаються напряму в ендпоінтах. Реалізовано фільтрацію фільмів за жанром.
5. **`main.py`** — Точка входу в додаток. Ініціалізує FastAPI, автоматично генерує таблиці при старті та описує API-ендпоінти, використовуючи `Depends(get_db)`.
